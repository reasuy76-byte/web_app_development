# 任務管理系統 PRD (Product Requirements Document)

## 1. 專案概述
- **背景與動機**: 現代人生活與工作忙碌，常有許多代辦事項需要處理。缺乏一個簡單且有效的系統來記錄與追蹤任務進度，容易導致遺漏或效率低下。本專案希望能提供一個輕量、直覺的任務管理平台，幫助使用者更有條理地管理日常任務。
- **目標用戶**: 需要個人任務管理工具的一般大眾、學生、或小型團隊成員。
- **核心價值主張**: 提供簡單易用的介面與操作流程，讓使用者能快速建立、追蹤並完成任務，減少認知負擔，專注於任務執行本身。

## 2. 功能需求
- **使用者故事一：任務建立與編輯**
  - 作為一般使用者，我希望能夠新增任務並設定標題與詳細描述，以便我完整紀錄每一個待辦事項。
  - 作為一般使用者，我希望能夠編輯現有任務，以便在事項內容變更時進行更新。
- **使用者故事二：任務狀態追蹤**
  - 作為一般使用者，我希望能夠在清單中快速切換或標示任務的狀態（例如：待處理、進行中、已完成），以便我追蹤最新的進度。
- **使用者故事三：任務清單檢視**
  - 作為一般使用者，我希望能夠在一個集中頁面看到所有任務列表，以便我掌握整體的待辦狀況與工作量。
- **使用者故事四：任務刪除**
  - 作為一般使用者，我希望能夠刪除已經不需要或不小心建立的任務，以便保持工作清單的整潔。
- **使用者故事五：任務過濾與篩選**
  - 作為一般使用者，我希望能夠根據任務的不同狀態來篩選清單，以便我能專注在當下需要處理的事項上。

## 3. 非功能需求
- **技術限制**: 
  - 後端框架：Flask (Python)
  - 前端渲染：Jinja2 (HTML/CSS)
  - 資料庫：SQLite
- **效能與安全考量**:
  - 系統應達到基本的回應速度需求，操作體驗不卡頓。
  - 需要對使用者的輸入進行驗證與過濾（防呆），以防範基礎的 XSS (Cross-Site Scripting) 與 SQL Injection 等安全風險。

## 4. MVP 範圍 (最低可行性產品)
- **Must Have (必須有)**
  - 建立任務（標題）。
  - 檢視全體任務列表。
  - 更新任務狀態（待完成、已完成）。
  - 刪除任務。
- **Should Have (應該有)**
  - 任務詳細描述欄位。
  - 依照狀態篩選清單視圖。
  - 修改任務標題與內容。
- **Nice to Have (加分項)**
  - 任務到期日期 (Deadline) 設定。
  - 使用者帳號與個人化列表。
  - 分類或標籤系統。
